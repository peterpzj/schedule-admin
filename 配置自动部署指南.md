## 一键配自动部署

### 步骤 1：生成服务器专属 SSH key（GitHub → 服务器）

这条密钥让 GitHub Actions 能登录你的服务器执行部署。

**方法 A：在服务器上生成（推荐）**

SSH 到你的服务器：
```bash
ssh user@your-server
```

服务器上执行：
```bash
# 生成专用于 GitHub Actions 的密钥
ssh-keygen -t ed25519 -C "github-actions-deploy" -f ~/.ssh/github_actions -N ""

# 显示私钥（要复制到 GitHub Secrets）
cat ~/.ssh/github_actions

# 显示公钥（要加到服务器 authorized_keys）
cat ~/.ssh/github_actions.pub >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

**方法 B：本地生成（如果你没服务器访问）**

我帮你本地生成 + 把公钥告诉你。

---

### 步骤 2：在 GitHub 仓库配置 Secrets

打开 **https://github.com/peterpzj/schedule-admin/settings/secrets/actions** → **New repository secret**

依次添加 3 个 secrets：

| Name | Value | 说明 |
|------|-------|------|
| `SERVER_HOST` | 你的服务器 IP（如 `43.138.100.50`） | 腾讯云公网 IP |
| `SERVER_USER` | SSH 用户名（通常是 `root` 或 `ubuntu`） | 用于登录的用户 |
| `SERVER_SSH_KEY` | 步骤 1 中的**私钥**全部内容（含开头 `-----BEGIN OPENSSH PRIVATE KEY-----`） | **保密！** |

---

### 步骤 3：触发首次部署

你只需要 git push 一次，GitHub Actions 会自动：
1. 收到 push 事件
2. SSH 到你的服务器
3. 执行 `cd /opt/schedule-admin && ./deploy.sh`
4. 服务自动拉代码、重新构建、重启

```bash
# 触发测试（你已经在 admin 目录）
cd admin
echo "测试自动部署" >> README.md
git add . && git commit -m "test: 触发自动部署" && git push
```

去 https://github.com/peterpzj/schedule-admin/actions 看到绿色 ✓ = 部署成功！

## 服务器端初次准备

服务器上先要：
```bash
# 1. SSH 到服务器
ssh user@your-server

# 2. 一次性运行（之前我给你写的脚本）
sudo bash <(curl -sSL https://raw.githubusercontent.com/peterpzj/schedule-admin/main/bootstrap.sh)
```

脚本会：
- 安装 Docker
- 克隆仓库到 /opt/schedule-admin
- 生成 .env
- 申请 SSL 证书
- 启动服务
